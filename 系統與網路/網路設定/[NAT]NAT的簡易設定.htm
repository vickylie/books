在回答這個問題之前，想先請您瞭解一下『虛擬 IP』與『實體 IP』的異同點，基本上，『虛擬IP』共有三段，而我們常使用的 192.168.xxx.xxx 就屬於 C Class 段的虛擬 IP 咯！那虛擬 IP 有何限制呢？基本上，虛擬 IP 是『不可以直接』跟 Internet 上面的資料交流的！所以，我們才需要 NAT 主機來將 虛擬 IP 的資料經過 IP 偽裝後，就可以與 Internet 溝通資料了！而由於這種『 IP 偽裝』使得經由 NAT 主機流出去的資料，看起來都是同一個 IP 的緣故！

OK！那我們來看看您的問題：
Windows ME 的設定是正確的！只不過在 DNS 與網域的設定上，可以選擇跟你最相近的主機名稱來填寫，不過，這個不重要！隨便寫也是可以的！
  1. 在 /etc/sysconfig/network 這個檔案裡面，將：
    GATEWAYDEV=eth0
    設定成為：
    GATEWAYDEV=ppp0
    或者根本就不要設定！因為網路會自動去捉取！
    另外，你可以在提示字元底下執行：
    echo 1 > /proc/sys/net/ipv4/ip_forward
    來啟動NAT的服務！而這一行應該要加入 /etc/rc.d/rc.local 裡面的最後一行，讓開機自動執行！

  2. 好了！剛剛我們提到所謂的虛擬 IP 不可以跟 Internet 直接溝通，所以就需要使用 IP 偽裝的技術了！要如何偽裝呢？如果你是以 Red Hat 6.x 以前的版本，那使用 ipchains 吧，如果是 RedHat 7.x 以後的版本，就需要特別注意了！因為 Red Hat 7.x 預設是同時會開啟 ipchains 與 iptables ，但這兩個套件會衝到！所以你必需關閉任何一個之後，才能進行 IP 偽裝術！
    執行 setup
    然後進入System services ，如果是 Red Hat 7.x 的話，那就關掉 ipchains 吧，然後：
    /etc/rc.d/init.d/xinetd restart
    啟動 iptables 吧！

  3. 開始偽裝了！你可以參考一下底下這一篇小州前輩文章：

    http://www.study-area.org/tips/iptables.htm

    或者直接在你的指令列下達：
    如果是 ipchains 的話，那就：
    ipchains -F
    ipchains -X
    ipchains -A forward -j MASQ -s 192.168.1.0/24 -d 0.0.0.0/0
    ipchains -P input ACCEPT
    ipchains -P output ACCEPT
    ipchains -P forward ACCEPT

    如果是 iptables 的話，那就：
    iptables -F
    iptables -X
    iptables -P INPUT ACCEPT
    iptables -P OUTPUT ACCEPT
    iptables -P FORWARD ACCEPT
    iptables -t nat -A POSTROUTING -o ppp0 -s 192.168.1.0/24 -j MASQUERADE

你先試試看吧！也可以看一下底下這一個關於網路除錯的網頁：
http://aerosol.ev.ncku.edu.tw/~vbird/problem/linux/problem_3.htm

是這樣的，如果你有看小州前輩寫的那篇關於 iptables 的 NAT 教學的話，那麼應該會知道其實還有一些模組需要載入的！例如底下的幾個模組：

modprobe ip_nat_ftp
modprobe ip_conntrack
modprobe ip_conntrack_ftp
modprobn ip_conntrack_irc

這些也都可以寫到 /etc/rc.d/rc.local 裡面去！其中，ip_nat_ftp就是在處理關於FTP的問題的！這個模組需要載入之後，你在進行 FTP 連線的時候，才不會有問題！

另外，你提到的關於 PASV 的開啟方法！呵呵！剛剛我才試出來！正在高興呢！如果你跟我一樣是使用 CuteFTP 的話，那麼在：最上方的命令列中的『ftp』『設定』裡面的『選擇』（註：每個版本或釵酗ㄕP，反正就是找到『設定』那一個選項就是了），然後尋找到『防火牆』的相關設定哪邊，你會看到需要輸入：
  1. 主機
  2. 埠
  3. 使用者
  4. 密碼
  5. 類型
  6. 啟動防火牆存取

等等的設定，將主機與埠號設定為你的 NAT 主機（例如 192.168.1.2 埠號為 21 啦）使用者與密碼就設定你的帳號與密碼，然後類型最重要，要選擇『PASV 模式』，然後在防火牆存取那邊打勾，就OK了！連上外部的 FTP 就沒問題啦！你可以試試看呦！加油吧！有問題再聯絡。
